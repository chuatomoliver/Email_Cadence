PAANO KO GINAWA ITONG EMAIL CADENCE APP (Para kay Boss)

Dito ko i-explain nang simple kung paano ko binuo 'tong system natin para madaling maintindihan.

1. AYOS NG MGA FILES (Monorepo)
Pinagsama-sama ko lahat sa isang "Monorepo" para hindi magulo. Ginamit ko ang Turbo para mabilis i-manage. Hati ito sa apat:
- Shared: Dito nakatago 'yung mga types na parehong gamit ng frontend at backend.
- API (NestJS): Ito 'yung nagsisilbing tulay. Dito natin sinesave 'yung mga cadence at dito rin naguutos sa Temporal na simulan na 'yung workflow.
- Worker (Temporal): Ito 'yung utak o makina sa likod. Sila 'yung gumagawa ng trabaho (pag-send ng email o pag-antay).
- Web (Next.js): Ito 'yung nakikita natin sa browser, 'yung may dashboard.

2. PAANO GUMAGANA 'YUNG WORKFLOW (Temporal.io)
Ginamit ko ang Temporal para sigurado tayong hindi titigil 'yung system kahit mag-error o mag-restart 'yung server.
- Sunod-sunod na steps: Ginagawa niya 'yung steps isa-isa (Send Email tapos Wait).
- Mock Email: Sa ngayon, naka-log lang sa console 'yung email (kunyari nag-send) para madaling i-test. Pero anytime, pwede natin 'tong kabitan ng totoong email provider.
- Silip Status (Query): Pwede nating silipin kung nasaan na 'yung workflow (Step 1 ba o 2) nang hindi ito naiistorbo.
- Update habang tumatakbo (Signal): Ito 'yung pinaka-astig. Kahit tumatakbo na 'yung workflow, pwede nating baguhin 'yung steps at itutuloy lang niya kung nasaan na siya.

3. ANG API AT FRONTEND
- API: Kumokonekta ito sa Temporal. Nilagyan ko rin ng "retry logic" para kung hindi pa gising 'yung server, mag-aantay lang siya.
- Frontend: Simple lang, may JSON editor para madaling baguhin 'yung cadence. Nag-tatanong din 'yung UI sa API bawat 2 segundo para laging updated 'yung progress bar sa screen natin.

4. MGA RULES SA PAG-UPDATE
Para hindi malito 'yung system kapag nag-update tayo:
- Kung tapos na 'yung step, hindi na uulitin.
- Kung nasaan na siya (index), doon pa rin siya itutuloy.
- Kung binawasan natin 'yung steps at lumampas na siya, matatapos na 'yung workflow nang kusa.

5. PAANO PAANDARIN
- May ginawa akong 'check-env.js' para i-check kung tama ba 'yung version ng Node at Temporal sa PC.
- Isang command lang: `npm run dev`, gagana na lahat nang sabay-sabay.

'Yun lang boss! Simple pero matibay 'yung pagkakagawa natin dito.
